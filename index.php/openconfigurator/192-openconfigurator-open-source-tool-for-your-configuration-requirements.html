openCONFIGURATOR – Open source tool for your POWERLINK configuration requirements






1. What software do I need?

Pre-requisites to install openCONFIGURATOR plugin

In Windows PC:

  • Java Runtime Environment v7 or higher – click here

  • Eclipse v3.8.1 or higher – click here

In addition to the above listed software, you have to install the boost libraries in your Linux PC to use openCONFIGURATOR plugin.

  • Download Boost libraries v1.58.0 or higher

    • sudo apt-get install libboost-all-dev

2. How to install and use openCONFIGURATOR

This section describes the steps on how to open the Eclipse IDE, install the openCONFIGURATOR plugin and create a new openCONFIGURATOR project using the plugin.

  • Open the Eclipse IDE and install the openCONFIGURATOR plugin following the steps below:

    • Go to "Help -> Install New Software" in Eclipse IDE

    • Copy and paste the update site URL - "http://sourceforge.net/projects/eclipse-plugin.openconf.p/files/update-site/" in "Work with" text box

    • Select the check box "Ethernet POWERLINK openCONFIGURATOR" under the "Industrial Automation" category and install the plugin

  • To install the openCONFIGURATOR plugin from Eclipse marketplace drag and drop this button to your running Eclipse workspace -

  • Give a project name and path, usually the projects are saved in the openCONFIGURATOR folder in the user directory. The network contains one Managing Node (MN) and several Controlled Nodes (CN). This plugin additionally supports Redundant Managing Node (RMN) configuration

  • The configuration for MN can be imported from the XDD/XDC file

  • Continue the setup by using the custom configuration. (If the configuration for the MN is imported using a XDD/XDC file then it is validated using schema validation)

  • Click "Finish"

  • In the project explorer, double click on the xml file, MN will be added in POWERLINK network view with the custom configuration

3. Adding a Redundant Managing Node to an existing openCONFIGURATOR project

POWERLINK supports high availability by allowing redundant masters in the network. At any point in time, only one master can be active and the others are in the standby mode. When the standby managing nodes detect a failure of the active master, one of them switch and take control of the network. The redundant masters are called as RMNs and the active master is called the AMN.

The following are the steps to create a redundant managing node in the network:

  • Open the existing openCONFIGURATOR project and right click on the Managing node and select Add node

  • Choose the node type as Redundant Managing Node and enter the "Node name" and "Node ID" and click on "Next"

  • Now you can see, RMN is configured with its custom configuration file

  • Click on "Finish"

  • Now the RMN will be added in the POWERLINK network view

  • Next build the openCONFIGURATOR project

4. Mapping of the PDO objects for the openCONFIGURATOR Controlled Node

In a network setup, some specific data needs to be transmitted frequently. Therefore In POWERLINK, Process Data Object (PDO) communication follows the publisher/subscriber principle to trasmit the data in the network during isochronous phase. 

The transmitting node creates a Transmit PDO (TPDO) that contains all actual values of a given set of objects and publishes it frequently to the network. On the receiving side, each node interested in one or several of the values can subscribe to this PDO, thus configuring it as a Receive PDO (RPDO). The process of assembling the TPDOs and RPDOs is called PDO Mapping. 

The PDO Mapping determines the payload of a PDO frame in a POWERLINK network. PDO payload is transported via the frame types PollRequest and PollResponse.

The following are the simple steps how to configure the parameters for the Controlled Node in the network:

  • Open the existing project in the openCONFIGURATOR plugin. Select MN/CN/RMN node in the POWERLINK network

  • Click "Show Mapping View" icon on the toolbar

  • PDO Configuration of mapping view summarizes the TPDO and RPDO channels. Let us now map the objects in the TPDO channel

  • Click on the cell of the “Mapped Object” column. A list of mappable objects will be shown in the drop-down list box

  • Select any of the object that you wish to map for that particular channel. Now the selected object is assigned to the corresponding channel index

  • There will be a corresponding increase in value (bytes) in the channel size text box. openCONFIGURATOR provides additional feature to "Map all available objects" and "Clear all objects"

  • Click on the "Map all available objects" button to map all the available TPDO/RPDO objects for the selected node and "Clear all" button enables you to clear the mapped objects

  • Similarly you can map the objects in RPDO channel

  • Objects with "Locked overlay" icon are non-editable objects

  • Now you can build your openCONFIGURATOR project

5. Parameter configuration for openCONFIGURATOR Controlled Node

Parameters define the size and structure of a mappable object or define actual values for objects depending on the IDE parameters. The "parameter configuration" is the process of configuring the parameter values by modifying certain parameters to configure the devices according to the requirement. This helps to configure certain parameters at the node level.

(Note: The XDDs used for configuration should support parameters)

This section describes you on how to configure the parameters for the Controlled Node in the network:

  • Open any existing openCONFIGURATOR project into the eclipse work space

  • Now, right click on the Managing node and select Add node. Provide a name for the Controlled Node and click "Next"

  • Now, Browse for a Controlled Node XDD which has parameter configuration inside them and click "Finish"

  • You can see the Controlled node is created in the POWERLINK network

  • Right click on the controlled node and select the "Show Object Dictionary"

  • It opens Object dictionary view where you can see "Switch to Parameter view" icon in the toolbar

  • Click on the "Switch to Parameter view" icon to view all the parameters which are available for the respective Controlled Nodes

  • To configure these parameters listed, right click on any one of the parameter and select "Properties"

  • Now you can see the properties listed out, choose advanced properties icon on the toolbar to view advanced properties of parameter

  • Now, right click on the Actual Value entry and you can configure the value from "off" to the available actual values which are listed down when you click on the drop down list

  • Now you can build the openCONFIGURATOR project

6. Modular device configuration for openCONFIGURATOR project

Modular device configuration allows a user to upgrade the devices easily by adding required modules to the existing system without having to use another separate device. Modular configuration is a way to configure the variable sub-modules such as I/O modules attached to the modular head device.

The following steps helps the user to add Modular devices in the network:

  • Open any existing openCONFIGURATOR project into the eclipse work space

  • Right click on the Managing node and select "Add node". Provide a name for the Controlled Node and click "Next"

  • Now, Browse for a Modular head Node XDD and click "Finish"

  • You can see that Modular head node along with an interface below is created in the POWERLINK network

  • The interface allows you to add modular child devices to the head node. Now, right click on the Interface and select "Add module"

  • Enter a name for the module and click Next. Browse for a child module XDD with respect to the module type which is displayed at the top and click Next

  • Another wizard appears with the details in which the position and address of the module will get placed. Like this you can create multiple modules according to your requirement

  • Now build the openCONFIGURATOR project
